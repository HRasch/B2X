## Agent-to-Agent Delegation Patterns

### 5 Core Patterns Documented

1. **Sequential (Waterfall)**
   - One agent's output feeds next
   - Example: Design â†’ Optimize â†’ Secure â†’ Complete
   - Use: Clear dependencies

2. **Parallel (Concurrent)**
   - Multiple agents work independently
   - Example: DDD + Performance + Security + Instrumentation
   - Use: Independent concerns

3. **Validation (Review Chain)**
   - Multiple agents validate quality
   - Example: Create â†’ Domain check â†’ Security check â†’ Approved
   - Use: Quality gates

4. **Escalation (Decision Making)**
   - Agents conflict â†’ escalate to main agent
   - Example: Performance vs. Accuracy â†’ @Architect decides
   - Use: Trade-offs

5. **Cascading (Progressive Refinement)**
   - Multiple rounds of refinement
   - Example: Design â†’ Apply constraints â†’ Optimize â†’ Validate
   - Use: Complex features
