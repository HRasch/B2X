# Guidelines Index

Dieses Verzeichnis enthält organisierte Guidelines für alle Aspekte der Projektverwaltung und Entwicklung, gepflegt von SARAH.

## Struktur

- `coding-standards/` - Code-Style, Naming Conventions, Formatierung
- `process/` - Workflow-Guidelines, Prozeduren, Best Practices
- `security/` - Security-Guidelines, Compliance-Anforderungen
- `quality/` - Testing-Standards, Code-Review-Kriterien
- `architecture/` - Architektur-Prinzipien, Design-Patterns
- `team/` - Team-Guidelines, Kommunikation, Zusammenarbeit

## Verwendung

Guidelines werden in kompakten, maschinenlesbaren Formaten dokumentiert:
- YAML für strukturierte Daten
- Markdown für lesbare Dokumentation
- JSON für programmatischen Zugriff

Agenten können diese Guidelines gezielt abrufen, um:
- Konsistent mit Projektstandards zu arbeiten
- Schnell auf etablierte Regeln zuzugreifen
- Automatisch Compliance zu prüfen
