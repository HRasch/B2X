# User Story Template

## User Story Format
**Als** [Benutzerrolle/Persona] **möchte ich** [Funktion/Aktion] **damit** [Business-Nutzen/Wert]

## Beispiel
**Als** eingeloggter Kunde **möchte ich** meine Bestellhistorie einsehen **damit** ich vergangene Käufe nachverfolgen kann

## Acceptance Criteria
**Gegeben** [Kontext/Initialer Zustand]  
**Wenn** [Aktion/Event]  
**Dann** [Erwartetes Ergebnis]

### Beispiele für Acceptance Criteria
- ✅ Bestellhistorie wird chronologisch absteigend sortiert angezeigt
- ✅ Jede Bestellung zeigt Datum, Gesamtbetrag und Status
- ✅ Klick auf Bestellung öffnet Detailansicht
- ✅ Performance: Liste lädt in < 2 Sekunden
- ✅ Mobile-responsive Design

## Definition of Ready (DoR)
- [ ] Business Value ist klar definiert
- [ ] Acceptance Criteria sind testbar formuliert
- [ ] Technische Machbarkeit wurde geprüft
- [ ] Abhängigkeiten sind identifiziert
- [ ] Story Points wurden geschätzt
- [ ] Priorität ist gesetzt (MoSCoW)

## Definition of Done (DoD)
- [ ] Code ist implementiert und reviewed
- [ ] Unit Tests sind geschrieben und bestehen
- [ ] Integration Tests sind implementiert
- [ ] Acceptance Criteria sind erfüllt
- [ ] Dokumentation ist aktualisiert
- [ ] Performance-Tests bestehen
- [ ] Security-Review wurde durchgeführt
- [ ] Feature ist deployed und funktional
