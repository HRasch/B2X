# Agent Support Request Process

## Wer kann SARAH um Unterstützung bitten?

Alle Agenten im Projekt können SARAH um Hilfe anfragen für:
- Guidance und Beratung
- Entscheidungshilfen
- Konflikt-Auflösung zwischen Agenten
- Alignment-Checks mit Projektstandards
- Permission-Anfragen
- Guideline-Clarifications

## Unterstützungstypen

### 1. Guidance Requests
**Wann:** Agent benötigt Rat oder Expertise
**Beispiel:** "SARAH, sollten wir diesen Ansatz oder jenen nehmen?"
**SARAH antwortet mit:** Empfehlungen, Best Practices, Trade-offs

### 2. Decision Support
**Wann:** Agent muss komplexe Decision treffen
**Beispiel:** "SARAH, wie treffe ich diese architektonische Entscheidung?"
**SARAH antwortet mit:** Analyse, Optionen, Empfehlung

### 3. Conflict Resolution
**Wann:** Zwei oder mehr Agenten sind uneins
**Beispiel:** "Backend und Frontend sind uneins über API-Design"
**SARAH antwortet mit:** Mediation, Ausgleich, finale Entscheidung

### 4. Standards Alignment Check
**Wann:** Agent fragt, ob etwas mit Standards konform ist
**Beispiel:** "Passt dieses Testing-Approach zu unseren Guidelines?"
**SARAH antwortet mit:** Ja/Nein mit Begründung und ggf. Verbesserungsvorschlägen

### 5. Permission Request
**Wann:** Agent benötigt Berechtigung für eine Aktion
**Beispiel:** "Darf ich diese Guideline aktualisieren?"
**SARAH antwortet mit:** Approval/Denial mit Begründung

## Request-Kanäle

1. **Direkt in Chat/Conversation**: "SARAH, benötige deine Unterstützung für..."
2. **Issue/Ticket-System**: Support-Request als formales Ticket
3. **Collaboration Channel**: In `.ai/collaboration/` dokumentieren

## SARAH's Response SLA

- **Guidance Requests**: Zeitnah, innerhalb Kontext
- **Decision Support**: Priorität basierend auf Urgency
- **Conflict Resolution**: Sofort/Priorisiert
- **Standards Alignment**: Schnell, mit Referenzen
- **Permission Requests**: Mit dokumentierter Begründung

## Documentation

Alle Support-Requests sollten dokumentiert werden in:
- `.ai/collaboration/support-log.md` - Support-Anfrage Log
- Relevante Entscheidungen in `.ai/guidelines/` aktualisieren
